# План фронтенда

Контекст и домен — в [booking-service.md](booking-service.md). Контракт — `contract/openapi.yaml`
(источник правды, не менять). Этот документ — про реализацию фронта.

## Стек

- React + Vite + TypeScript, пакетный менеджер npm.
- API: типы из контракта через `openapi-typescript`, запросы через `openapi-fetch`
  (типобезопасный fetch-клиент). Скрипт `npm run gen:api` генерит типы из
  `contract/openapi.yaml`.
- Даты/таймзоны: `date-fns` + `date-fns-tz` (или нативный `Temporal`).
- Тесты: Vitest + React Testing Library + MSW.
- Сгенерённые типы API **не коммитим** (генерятся скриптом), добавить в `.gitignore`.

## Экраны

**Публичный (посетитель)** — по публичной ссылке:
- Месячный календарь (сетка) с двумя стрелками переключения месяца; за горизонтом
  стрелки неактивны; прошлые дни серые/неактивные.
- Подсветка дней, где есть свободные слоты.
- Клик по дню → список слотов дня (свободные/занятые; занятые — без указания кем).
- Клик по свободному слоту → форма брони: имя (req), ссылка на встречу (req, URL),
  описание (opt). Валидация на стороне фронта (+ обработка ошибок бэка).
- После брони слот становится занятым и недоступным (защита от дабл-сабмита — на фронте).
- Явно показывать, в каком часовом поясе живёт владелец.

**Владелец (admin)** — с токеном (хранить в `localStorage`, слать `Authorization: Bearer`):
- Тот же месячный календарь, подсветка дней со встречами.
- Клик по дню → слоты; у занятых видны детали (имя, ссылка, описание).
- Отмена брони.
- Где-то на экране — публичная ссылка на профиль для копирования.

## Время на фронте

- Конфиг (`GET /api/v1/config`) тянем один раз на старте: `ownerTimeZone`, рабочие часы и т.д.
- Слоты приходят как `OffsetDateTime`; показываем в TZ браузера, но подписываем, что
  владелец живёт в `ownerTimeZone` (напр. «Время владельца: Asia/Yerevan (UTC+4)»).
- Календарную сетку строим по дням; границы дней согласованы с тем, что отдаёт бэк.

## Ошибки

Парсим `application/problem+json` (RFC 9457): по `errorCode` показываем понятные
сообщения; для валидации — мапим `errors[]` на поля формы. 409 (слот занят) → показать,
что слот только что заняли, обновить день.

## Тесты (Vitest + RTL + MSW)

- MSW мокает API строго по контракту (фронт пишется без живого бэка).
- Покрыть: рендер календаря и подсветку, дизейбл прошлого/за-горизонта, открытие дня,
  форму брони и её валидацию, обработку ошибок (409/422), admin-вид с деталями и отмену.

## Сборка и запуск

- `npm ci && npm run gen:api && npm run dev` — локальная разработка.
- `npm run test` — Vitest. `npm run build` — продакшен-сборка в `dist/`.
- `dist/` на этапе Docker-сборки копируется в статику Spring.

## Definition of Done (фронт-сессия)

- Оба экрана (посетитель и владелец) работают на MSW-моках по контракту.
- Vitest зелёный.
- `npm run build` проходит без ошибок.
