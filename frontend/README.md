# Фронтенд проекта «Запись на звонок»

React + Vite + TypeScript приложение для бронирования звонков.

## Стек

- **React 19** + **Vite** + **TypeScript**
- **Tailwind CSS** - стилизация
- **openapi-typescript** + **openapi-fetch** - типобезопасный API клиент
- **date-fns** + **date-fns-tz** - работа с датами и часовыми поясами
- **Vitest** + **React Testing Library** - тестирование
- **React Router** - навигация

## Установка и запуск

```bash
# Установка зависимостей
npm ci

# Генерация API типов из контракта
npm run gen:api

# Запуск dev-сервера
npm run dev

# Запуск тестов
npm test

# Продакшен сборка
npm run build
```

## Экраны

### Посетитель (`/`)
- Месячный календарь с навигацией
- Подсветка дней со свободными слотами
- Просмотр слотов выбранного дня
- Бронирование свободного слота (форма с именем, ссылкой на встречу, описанием)
- Показывает часовой пояс владельца

### Владелец (`/admin`)
- Авторизация по Bearer токену (сохраняется в localStorage)
- Календарь встреч и список предстоящих встреч в две колонки на десктопе, стеком на мобильных
- Индикатор количества броней расположен справа внутри занятого дня
- Просмотр имени, ссылки и описания для каждой брони
- Отмена броней
- Копирование публичной ссылки

## API

Все типы генерируются из `contract/openapi.yaml` через `openapi-typescript`.

Клиент использует `openapi-fetch` для типобезопасных запросов:
- Публичные запросы: `apiClient`
- Админские запросы: `adminClient` (автоматически добавляет `Authorization: Bearer <token>`)

## Время

- Конфигурация загружается при старте: `ownerTimeZone`, рабочие часы, длина слота, горизонт
- Слоты приходят как `OffsetDateTime`, отображаются в локальном времени браузера
- Календарная сетка строится по дням в часовом поясе владельца

## Тесты

Vitest + React Testing Library:
- Изолированные тесты компонентов
- MSW моки для API (по контракту)

## Сборка

Продакшен сборка создаёт `dist/` директорию, которая копируется в Spring Boot приложение для отдачи статики.
